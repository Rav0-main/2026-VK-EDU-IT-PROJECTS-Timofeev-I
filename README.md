# Домашнее задание №1

## Идея

**"Личный" менеджер по отслеживанию питания**.

## Elevator Pitch

Большинство людей в мире страдает ожирением.\
Одно из возможных решений - это составление и оценка питания.\
В отличие от других приложений, мы учитываем всевозможную информацию о пользователе: его локацию, физическую активность и потребление продуктов. В результате клиент получает исчерпывающие рекомендации.

## Lean Canvas

### 1. Сегменты потребителей.

* Люди, имеющие проблемы со здоровьем или желающие следить за своим здоровьем.
* Профессиональные спортсмены или любители.
* Врачи и специалисты, занимающиеся рекомендациями питания.
* Сети питания для рекламы их в приложение.
* Рекламодатели.

### 2. Проблема.

* Неудобство отслеживания питания повсеместно.
* Поиск клиентов для врачей и специалистов по здоровью.
* Поиск заведений с меню правильного питания.

### 3. Уникальная ценность.

**Предложение**:
> Проверь, что и где можно поесть, не вредив своему здоровью.

**Конкуренты**:
* Самозанятые эксперты по питанию.
* Приложения по статистике о здоровье: Samsung health, Apple health и т.д.

### 4. Решение.

* Добавление нейронных сетей для быстрых рекомендаций. Условно, сфоткал продукт (котлета с картофильным пюре) - получил калорийность и рекомендации о его частоте потребления.
* Разрешаемый просмотр о покупаемых продуктов для информации о потребляемой еды и предупреждениях, если товар не соответсвует плану.
* Поиск нужных заведений питания.
* Способность закрепиться в системе и иметь быструю связь с экспертами по питанию.

### 5. Каналы.

* Информационные поля, специализирующиеся на здоровье: телеграм-каналы, сайты, видеокреаторы.
* Составление рекламы на основе предоставленной пользователем информации.
* Рестораны, столовые и т.д.

### 6. Поток прибыли.

* Улучшение качества за счёт дополнительной платы:
    * Расширение возможностей для пользователей, которые отслеживают своё питание.
    * Улучшение рекомендаций для экспертов по питанию и заведений по питанию.

* Плата от рекламодателей - предоставление разрешаемой информации пользователя.
* Компании, предоставляющие своим работникам питание за свой счет.

### 7. Структура издержек.

* Эксплуатация и поддержка системы.
* Реклама для компаний, пользователей и экспертов по питанию.

### 8. Ключевые метрики.

* Количество экспертов на количество пользователей.
* Частота полезных советов, приходящих к пользователю.
* Количество положительно/отрицательно оцененных рекомендаций: экспертов по питанию и сетей питания.
* Количество и время заходов в приложение в день.

### 9. Скрытое преимущество.

* Полнота - не только количество шагов в день.
* База зафиксированных (по своему желанию) экспертов и пользователей.
* Чем больше пользователь использует приложение, тем лучше рекомандации для него.
* Чем больше качественных рекомендаций, тем больше устоявшихся пользователей: нежелание переходить на аналоги.
